# CFD 妥当性レビュー対応計画

**レビュー日**: 2026-04-06
**レビュー元**: 外部 LLM による客観的評価
**対応方針**: 全3指摘を受け入れ、優先順位を再構成

---

## レビュー指摘の要約

### 評価された点
- ソフトウェアアーキテクチャ（ハーネス駆動、テスト体制）は堅牢
- 簡易版の初期場生成アプローチは有効

### 指摘された課題
1. **「正解」不在の相互比較** — 実験データなしに 簡易版 vs OpenFOAM を比較しても物理的正しさは保証できない
2. **G_b 無効化による成層弱化** — 浮力生成項なしのサウナ CFD は無意味
3. **メッシュ・BC の妥協** — M0 (9,600 cells) + fixedValue 壁面では熱収支が崩れる

**総評**: 「計算を回すこと」から「物理的に正しい解を得ること」へのフェーズ移行が必要

---

## 対応計画

### Phase 1.5: OpenFOAM 物理的妥当性の確保（最優先）

#### 1-A: G_b 有効化
- **方法**: ケースを Linux FS (`~/openfoam_cases/`) で全工程実行
- **代替案**: codedSource → 既存 OpenFOAM ソース (`buoyantKEpsilon` の G_b 実装を参考に) で fvOptions.C をプリコンパイル
- **完了条件**: G_b ありで 300s 計算が安定動作

#### 1-B: 壁面 BC の正常化
- `externalWallHeatFluxTemperature` (mode=coefficient) に統一
- 初期不安定性対策:
  - deltaT = 0.0001 から開始（現在の 0.001 より 10 倍小さく）
  - `nOuterCorrectors = 4`（現在 2 → 増やして収束を保証）
  - 簡易版初期化で壁面温度が整合済み（55°C スタート）
- **完了条件**: externalWallHeatFlux で 300s 発散なし

#### 1-C: メッシュ M1 への移行
- M0 (9,600) → M1 (76,800) に変更
- メッシュ依存性確認: M0 と M1 の温度プロファイル比較
- **完了条件**: M0→M1 で温度差が ±10% 以内に収束

### Phase 1.5 の完了基準
- G_b 有効 + externalWallHeatFlux + M1 メッシュで定常到達
- 温度成層 ΔT が物理的に妥当（20-40K 範囲）
- 残差が十分に低下（U, h < 1e-3）

---

### 実験データの前倒し取得 (Phase 4 一部前倒し)

#### 最低限のデータセット
- ドライサウナ定常状態: 3点の温度プローブ（上段ベンチ、下段ベンチ、床面）
- 計測方法: 熱電対 or サーミスタ、30分間の定常温度
- 条件: ヒーター出力既知、壁面材質既知、部屋寸法既知

#### データなしでの代替
- 文献データ（Kukkonen-Harjula et al., ISO 24553 サウナ温度プロファイル）
- メーカーカタログの実測温度範囲

---

### Phase 2/3 への移行条件

以下を**全て**満たした場合にのみ Phase 2 (ロウリュ) / Phase 3 (アウフグース) に進む:

1. OpenFOAM ドライ定常状態が実験データ（or 文献値）と許容誤差内で一致
2. メッシュ依存性が確認済み（M1 → M2 で差が小さい）
3. G_b 有効 + externalWallHeatFlux で安定動作
4. 簡易版の定常解が OpenFOAM と定性的に一致（成層方向、界面位置）
