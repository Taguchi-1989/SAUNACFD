**作成日**: 2026-04-20  
**計算条件共通**: buoyantSimpleFoam, M0 mesh (76,800 cells), viewFactor radiation, 50,000 iterations

---

## 1. 目的

前セッション (Case I, 壁厚 0.015m) で upper_bench 111°C (実測 80-100°C) という結果を得た。
本セッションでは以下の3つのパラメータを変動させ、実測温度への接近を図る:

1. **壁厚**: 0.015m → 0.08m (wall_htc: 8.0 → 1.5 W/(m²K))
2. **ヒーター壁温 (fixedT サロゲート)**: 473K / 573K / 673K
3. **換気**: supply_exhaust (~5 ACH) の有無

## 4. 分析と考察

### 4.1 ヒーター壁温の影響は極めて小さい

- 473K → 573K → 673K で upper_bench は 234-235°C でほぼ不変
- heater_wall 放射は 214W → 1,161W → 2,651W と変化するが、13,000W volume source に対して2-20%程度
- **結論**: viewFactor 放射の fixedT サロゲートは温度場にほとんど影響しない。放射は heater→wall 方向に主に作用し、室内空気温度への直接効果は限定的

### 4.2 壁厚 0.08m でも温度過大 (Case I 系)

- 前セッション (壁厚 0.015m, wall_htc=8.0) では upper 111°C
- 今回 (壁厚 0.08m, wall_htc=1.5) では upper **235°C** — 逆に上昇

**原因分析**: wall_htc の低下は外部への放熱抵抗の **増加** を意味する:
- wall_htc = conductivity / thickness = 0.12 / 0.08 = 1.5 W/(m²K)
- これは externalWallHeatFluxTemperature BC の `h` パラメータ
- h が小さい = 壁の断熱性が高い = **室温が上昇** する

**前セッションとの矛盾**:
- 前セッション Case I (壁厚 0.015m, 30k iter) で 111°C だったのは iter 不足で未収束だったため
- 50k iter まで延ばすと 235°C まで上昇 = steady state に近づくにつれ温度上昇

### 4.3 換気が支配的な冷却メカニズム (Case J)

- 換気追加で upper 235°C → **67°C** (実測 80-100°C の下限に接近)
- ただし lower_bench 29°C は実測 40-60°C より低い
- 換気量が過大の可能性 → area 調整で最適化可能

### 4.4 Steady state 収束状況

- Case I 系: rho min 1.01 → 空気密度が高温で大きく変化、50k iter でプローブ温度は振動 (±2K)
- Case J: rho min 0.62 → 高温領域は limited、温度は安定 (振動 ±1K)
- **Case J は概ね steady state に到達**

## 5. 次のアクション

### Priority 1: 換気量の調整 (Case J ベース)

Case J の upper=67°C を 80-100°C に引き上げるための選択肢:

| パラメータ | 現在値 | 提案値 | 期待効果 |
|-----------|--------|--------|---------|
| supply area | 0.01 m² | 0.005 m² | ACH半減、室温上昇 |
| exhaust area | 0.015 m² | 0.008 m² | 排気量低下 |
| heater power | 13 kW | 16 kW | 入力増加 |

### Priority 2: wall_htc の再検討

現在のモデル: `h = lambda / delta` は「厚い壁 = 断熱性が高い」を正しく表現している。
しかし実際のサウナは：
- 内壁面: 木材パネル (厚 0.02m, lambda=0.12)
- 断熱材: ミネラルウール (厚 0.06m, lambda=0.04)
- 外壁: 構造材

等価 U-value = 1 / (0.02/0.12 + 0.06/0.04 + 外部対流抵抗)
           = 1 / (0.167 + 1.5 + 0.1) ≈ 0.57 W/(m²K)

→ wall_htc を 0.57 に設定すれば更に断熱的 (温度上昇方向)
→ 換気が主要な冷却パスであることを裏付け

### Priority 3: volume source 出力の検討

13kW volume source は空気を直接加熱 → 235°C は物理的に非現実的。
実際のヒーター:
- 石に熱を蓄え → 対流+放射で室内へ
- 石の表面温度 300-400°C、空気接触面積が限定的
- 有効対流伝熱 ≈ 5-8 kW (残りは放射+石蓄熱)

**提案**: volume source を 5-8 kW に低減 + fixedT 放射を維持

## 6. 結論

| 知見 | 詳細 |
|------|------|
| ヒーター壁温パラメトリック | 473K-673K の範囲で室温への影響は無視可能 (<1°C) |
| 壁厚増加の効果 | 断熱性向上 = 温度上昇方向 (当初の予想と逆) |
| 換気が決定的 | 換気なし 235°C → 換気あり 67°C (Δ168°C) |
| 実測値への最適経路 | Case J ベースで換気量微調整 + volume source 出力見直し |

**最有望ケース**: Case J (viewFactor + 換気) をベースに、supply/exhaust area を半減させたバリアントを試行。
