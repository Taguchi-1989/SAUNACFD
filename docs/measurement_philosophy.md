# サウナ室計測フィロソフィー — SaunaFlow CFDバリデーション

## 1. 何を検証するのか

SaunaFlowのCFDシミュレーションが出力する**温度成層化**が、実際のサウナ室で再現されているかを確認する。

CFDが予測する主要な現象:

| 現象 | KPI | CFD予測値 (Case B) | 検証に必要な計測量 |
| --- | --- | --- | --- |
| 上下温度差（成層化） | K-01 | upper 108°C, lower 68°C, ΔT≈40K | 温度×2点以上 |
| 温度の鉛直分布 | K-07 | (upper-lower)/mean ≈ 0.12 | 温度×2点以上 |
| 定常温度の絶対値 | - | upper≈108°C, lower≈68°C | 温度（校正済み） |
| ロウリュ時の湿度ピーク | K-03 | humidity peak | 湿度（Phase 2） |
| ロウリュ時の温度変化 | K-02, K-04 | peak rise, arrival time | 温度時系列（Phase 2） |

**Phase 1の検証目標**: 定常状態における lower_bench 温度1点を、CFD予測と比較する。
「1点でも実測が入る」ことがバリデーションの最初の一歩。

---

## 2. なぜ「投入過渡」も記録するのか

センサーを予熱済みサウナ室に投入すると、室温（~20°C）からサウナ温度（~68-110°C）まで温度が上昇する過渡過程が記録される。この過渡データは Phase 1 のバリデーション対象ではないが、以下の理由でバックデータとして保持する:

1. **センサー応答特性の把握**: DHT22 の時定数（どのくらいの速さで追従するか）がわかる。定常到達判定の根拠になる。
2. **定常到達の判定**: 温度変化率 dT/dt が十分小さくなった時点を「定常」とみなす。この判定には過渡データが必要。
3. **将来のtransient検証への布石**: Phase 2以降で buoyantPimpleFoam の過渡計算と比較する際、センサー投入後の昇温カーブ自体が検証データになりうる。
4. **再現性の確認**: 複数セッションの過渡カーブを比較し、計測条件のばらつきを評価できる。

**実運用**: 記録は常に時系列全体を取る。validation には定常区間のみ使う（既存 cli.py の「後半50%平均」ロジック）。

## 3. センサー校正の考え方

### 3.1 なぜ校正が必要か

DHT22 の公称精度は ±0.5°C（-40~80°C）だが:

- 個体差がある（同一ロットでも ±1°C 程度のばらつき）
- 高温域（60-80°C）では精度が落ちる（仕様範囲ぎりぎり）
- 配線延長やサウナ環境の影響で追加の誤差が入る可能性

CFDバリデーションの許容差は ±3-5°C だが、「±5°C の中のどこにいるか」を知ることで、シミュレーション改善の方向性がわかる。

### 3.2 校正手順

```text
Step 1: 氷水点 (0°C)
  氷水混合物にセンサーを浸漬
  DHT22読み値を記録
  オフセット = 0 - 読み値

Step 2: 沸騰点 (100°C) ※DHT22は80°C上限のため省略可
  代替: 60-70°Cの温水で中間点チェック
  温度計（別の基準温度計）と比較

Step 3: 相互校正
  複数DHT22を同一環境に設置
  個体差を記録 → meta.yaml に補正値として保存
```

### 3.3 校正定数の管理

校正は「センサーの個性」を知る作業。結果は `session_meta.yaml` に記録:

```yaml
calibration:
  date: "2026-04-20"
  sensor_id: "DHT22-001"
  ice_point_offset_c: -0.3    # 0°Cで -0.3°C読み → +0.3補正
  reference_check_c: 65.0     # 基準温度計 65.0°C
  sensor_reading_c: 64.2      # DHT22読み 64.2°C
  correction_applied: true
```

## 4. 定常到達の判定

### 4.1 問題

「定常」はいつ来るのか？サウナは常にゆらいでいる（対流、扉の開閉、ヒーター制御の on/off サイクル）。完全な定常はない。

### 4.2 実用的な定常判定

```text
判定基準: 直近N秒間の温度変化率 |dT/dt| < ε

推奨値:
  N = 60秒（1分間の移動平均）
  ε = 0.1°C/min（6秒あたり ~0.01°C）
```

DHT22 のサンプリング間隔は2秒、分解能は0.1°C。したがって:

- 「60秒間で温度が0.1°C以上変わらなければ定常」が実用的
- この判定はホスト側 (src/daq/) で行い、meta.yaml に定常到達時刻を記録

### 4.3 バリデーションとの関係

既存の cli.py:174 は「時系列の後半50%」を定常区間とみなして平均する。

これは「十分長く測定すれば後半は定常」という仮定。Phase 1 では:

- 投入後 5-10 分で定常到達を確認
- その後 10 分以上の定常記録を取る
- 合計 15-20 分のセッション → 後半50%は定常区間に収まる

## 5. DHT22 の限界と Phase 2 への橋渡し

### 5.1 DHT22 で測れるもの（Phase 1）

| 計測量 | 設置位置 | 精度 | 備考 |
| --- | --- | --- | --- |
| 温度 | lower_bench (y=0.8m, ~68°C) | ±1°C (校正後) | 仕様範囲内 |
| 相対湿度 | lower_bench (y=0.8m) | ±2-5%RH | 高温域で精度低下 |

### 5.2 DHT22 では測れないもの

| 計測量 | 理由 | Phase 2 での対策 |
| --- | --- | --- |
| upper_bench 温度 (80-110°C) | DHT22上限80°C超過 | K型熱電対 + MAX31855 |
| 高精度湿度 | DHT22の高温域精度不足 | SHT-85 (上限130°C) |
| 風速 | DHT22は温湿度のみ | ホットワイヤ式風速計 |
| 速い過渡応答 | DHT22時定数が長い | 熱電対（応答 ~ms） |

### 5.3 ロウリュ計測への展望（Phase 2）

ロウリュ（蒸気投入）時に期待される現象:

1. **湿度急上昇** → 絶対湿度 [kg/kg] のピーク（K-03）
2. **温度上昇加速** → 水蒸気の潜熱放出で空気温度が上がる（K-02）
3. **ピーク到達時間** → 投入から何秒でピークに達するか（K-04）

DHT22 のサンプリング2秒・応答遅れを考えると、K-04（到達時間 ±5-10秒の精度）は限界ぎりぎり。
Phase 2 では応答の速い熱電対と高温対応湿度センサーで過渡を取る。

## 6. 計測セッション設計

### 6.1 Phase 1 セッションプロトコル

```text
T-30min  サウナ予熱開始（ヒーター ON）
T-5min   センサー校正確認（室温ベンチでクイックチェック）
T=0      センサーをサウナ室に投入、記録開始
T+0~10min  過渡区間（バックデータとして記録）
T+10min  定常到達判定 → 以降が定常区間
T+10~25min  定常記録（validation用、最低10分）
T+25min  記録停止、センサー撤収
```

合計セッション: 約25分。うち定常区間15分 → 後半50%で7.5分の平均値を取得。

### 6.2 データ量見積

- DHT22: 2秒サンプリング → 25分で750サンプル
- CSV行: 約40バイト/行 → 750 x 40 = 30KB/セッション
- 保存: MicroSD/フラッシュに十分すぎる

### 6.3 セッション間の再現性

同一条件で複数セッションを実施し、ばらつきを確認する:

- 最低3セッション（同一日の異なる時間帯、または異なる日）
- セッション間の定常温度差が ±2°C 以内なら再現性あり
- ±5°C 以上のばらつきがあれば、計測条件の見直し（設置位置、サウナ状態など）

## 7. 絶対湿度変換の前提条件

### 7.1 変換式

```text
相対湿度 %RH → 絶対湿度 w [kg vapor / kg dry air]

w = 0.622 x Pv / (Patm - Pv)
Pv = (RH/100) x Psat(T)
Psat(T) = Antoine equation
```

### 7.2 前提の固定

| パラメータ | 値 | 理由 |
| --- | --- | --- |
| 大気圧 Patm | 101325 Pa | 標準大気圧。日変動 ±1-2% は湿度換算への影響が小さい |
| Antoine定数 | NIST標準値 | 水の飽和蒸気圧。100°C以下と以上で係数切替 |

- 計測日の気圧は meta.yaml に記録（将来の補正オプション）
- K-03 は kpi.py で `kg/kg x 1000 -> g/kg` 変換を期待

---

## 8. まとめ: Phase 1 → Phase 2 の判断基準

Phase 1 が成功したと判断する条件:

1. lower_bench で定常温度が取得できている（dT/dt < 0.1°C/min）
2. `saunaflow report` が正常実行され、ValidationReport が生成される
3. CFD予測との差が ±30K 以内（現在の許容差）
4. 3セッション以上で再現性 ±2°C 以内

上記を満たしたら Phase 2（upper_bench 用 K型熱電対、ロウリュ計測）に進む。
